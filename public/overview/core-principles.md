# Core principles

1. **Sequential access (gates)** — Users progress Infra → Data → Context → AI → Execution → Capital → Inner. Capability accumulates; permission follows behavior + data + capital + AI usage.

2. **Capability > permission** — Access tiers (Public · Lens · Verified · Institutional · Elite) derive from on-chain registry + gate scoring, not static allowlists alone.

3. **Inner vs outer store** — Public chain state is outer; personalized modules (flow maps, AI terminals, strategy rooms) load via `clrty_inner_enter` when capability thresholds are met.

4. **Execution-aware system** — Settlement and routing respect live liquidity, HELIX simulation, and deterministic finality targets (<400ms language on public site — aspirational SLA, verify per environment).

5. **Platform over token event** — 16M fixed supply, validator/LP economics, and long-horizon coordination — not launch-week hype mechanics.
